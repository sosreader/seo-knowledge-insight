## Context

使用者每週與 SEO 顧問（Gene Hong）開會，需要：
1. 不只看數字，還要帶著**有深度的問題**去開會
2. 交叉比對**資料庫 QA + 顧問文章觀點 + 當週指標異常 + 最新業界動態**，找出矛盾與缺口
3. 參考業界/學術框架，讓分析有結構、有依據
4. 與現有 `/generate-report` 週報互補（不取代）
5. **每次執行的分析成果要留存**，回饋到 `research/` 知識體系，為後續演進累積

現有 `/generate-report` 回答「本週 SEO 怎麼了？」，新的 `/meeting-prep` 回答「**我該在會議上問什麼？為什麼？**」

**「Claude Code as LLM」模式**：Claude Code 本身就是語意判斷引擎，直接讀取資料 + 推理生成報告。不呼叫 OpenAI 或任何外部 LLM API。WebFetch/WebSearch 是 Claude Code 的 built-in tools，用來取得最新網路資料，不算外部 LLM 呼叫。

## 決策

### 為何新 command 而非擴充 `/generate-report`

- 用途不同：週報（閱讀/歸檔） vs. 會議準備（對話/核查）
- `generate-report.md` 已 339 行，加深度研究分支會破 800 行上限
- 典型工作流：先跑 `/generate-report` → 再跑 `/meeting-prep --report <路徑>`

### 網路研究：自動化，不留給使用者

SEO 沒有新資料開會沒意義。Command 執行時**自動抓取最新業界動態**：

| 優先序 | 來源 | 方式 | 為何選它 |
|--------|------|------|---------|
| P0 | Google Search Status Dashboard | WebFetch JSON (`status.search.google.com/incidents.json`) | 官方、結構化、即時 |
| P1 | Search Engine Roundtable | WebFetch HTML（最新 5 篇摘要） | 最快的 breaking news |
| P2 | WebSearch 動態查詢 | 用 ALERT_DOWN 指標名搜尋 `"<指標> site:searchengineland.com OR site:developers.google.com 2026"` | 針對異常指標找最新報導 |

Claude Code command 環境中 WebFetch/WebSearch 是安全的（使用者本機，非 API server）。

### 文件留存策略

每次 `/meeting-prep` 執行後，自動將可累積的洞察寫入 `research/`：

| 產出 | 留存位置 | 說明 |
|------|---------|------|
| 本次深度報告 | `output/meeting_prep_YYYYMMDD_{hash8}.md` | 完整輸出，含 citations |
| 業界動態摘要 | `research/11-seo-industry-updates.md`（新增） | 累積式，每次 append 一個 section |
| E-E-A-T + 成熟度評分歷史 | `research/12-meeting-prep-insights.md`（新增） | 追蹤評分變化趨勢 |
| 顧問文章觀點索引 | `research/12-meeting-prep-insights.md` | 交叉比對中發現的矛盾/一致，累積 |

**演進路線**（後續 iterations）：
- v1（本次）：command 生成報告 + 自動 append research/
- v2：會議後紀錄回饋（`/meeting-debrief`），將顧問實際回答寫回 KB
- v3：歷史趨勢分析 — 比較多次 meeting-prep 的評分變化、問題解決率

## 資料流

```
INPUT（三選一）
  A: snapshot JSON 路徑
  B: Google Sheets URL
  C: --report output/report_*.md（shortcut，跳過指標解析）
      ↓
STEP A: 取得異常摘要 → ALERT_DOWN 清單
      ↓
STEP B: 網路研究（並行 3 路）
  ├─ WebFetch: Google Search Status Dashboard JSON（最近 30 天）
  ├─ WebFetch: Search Engine Roundtable（最新 5 篇）
  └─ WebSearch: 每個 ALERT_DOWN 指標 × "site:searchengineland.com 2026"
  → industryMap（10-15 筆動態）
      ↓
STEP C: 多主題 KB 搜尋（7-10 輪）→ qaMap（20-30 筆）
      ↓
STEP D: 顧問文章定向讀取（Grep + Read，最多 3 篇 × 120 行）
      ↓
STEP E: Claude Code 分段推理 → 11 sections
      ↓
STEP F: 存檔
  ├─ output/meeting_prep_YYYYMMDD_{hash8}.md（完整報告）
  ├─ research/11-seo-industry-updates.md（append 業界動態）
  └─ research/12-meeting-prep-insights.md（append 評分 + 洞察）
```
