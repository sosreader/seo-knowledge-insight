## Overview

L4（領先期）內容僅 36 筆（2.4%），不足以服務 L4 客戶的 chat/search/meeting-prep 需求。三管齊下：reclassify 低估的 L3 項目、從現有文章重萃 L4 洞察、為 6 個缺口主題生成新 Q&A。目標：L4 從 36 → 80+（5-7%）。

## Phase 1：Classifier 強化 + L3 候選審計

**目的**：擴充 L4 keyword set，找出被低估的 L3 項目。

### Step 1：擴充 classifier L4 keywords
- **檔案**：`utils/maturity_classifier.py`
- **動作**：L4_KEYWORDS 加入中文 variants：`自動化測試`、`競爭情報系統`、`跨通路整合`、`歸因模型`、`增量價值`、`知識圖譜`、`排名預測`、`內容生成 pipeline`
- **風險**：低（additive，existing tests 覆蓋）

### Step 2：更新 golden dataset
- **檔案**：`eval/golden_maturity_classification.json`
- **動作**：+4 L4 edge cases（中文預測模型、跨通路歸因、L3 邊界case、自動化測試框架）
- **30 → 34 cases**

### Step 3：L3 候選審計
- **動作**：對所有 136 筆 L3 項目重跑 updated classifier，找出升級為 L4 的候選
- **Claude Code 作為 judge**：逐一審查候選項，確認答案描述的是「如何實作」預測/跨通路系統，不只是提到術語
- **預期產出**：15-25 筆 reclassification

### Step 4：Classifier eval 驗證
- **指令**：`python evals/eval_maturity_classification.py`
- **Gate**：accuracy >= 0.80, adjacent >= 0.90

## Phase 2：現有文章 L4 深度挖掘

**目的**：從 genehong-medium（21 筆 L4 來源）和 seo-meetings 重讀，萃取遺漏的 L4 洞察。

### Step 5：辨識高信號來源檔案
- **動作**：掃描 `raw_data/medium_markdown/` 和 `raw_data/markdown/` 中含 L4 關鍵字的檔案
- **預期**：10-15 篇高潛力文章

### Step 6：L4 聚焦重萃
- **動作**：Claude Code 直接讀取每篇文章，用 L4 視角萃取 Q&A
- **輸出**：`output/qa_l4_extracted/*.json`
- **品質標準**：答案 >= 150 字、含具體工具/指標、keywords 含 L4 術語
- **預期**：20-30 筆淨新 L4

## Phase 3：6 缺口主題 Q&A 生成

**目的**：填補現有知識庫完全缺失的 L4 主題。

### Step 7：建立 seed context
- **動作**：為 6 個主題各建 seed markdown（2 段背景 + 5-8 問題提示）
- **路徑**：`raw_data/l4_seeds/*.md`

### Step 8：生成 Q&A
- **動作**：Claude Code 對每個 seed 生成 5-8 筆 Q&A
- **格式**：標準 QA schema + `source_collection: "l4-seeds"`、`maturity_relevance: "L4"`
- **輸出**：`output/qa_l4_generated/*.json`
- **預期**：30-48 筆

### Step 9：品質審查
- **4 維度評分**：具體實作方法（非泛泛而談）、真實顧問場景、與現有 L3 差異化、keywords 對齊
- **Gate**：score <= 2 拒絕，目標接受率 70%+

## Risks & Mitigations

| 風險 | 影響 | 緩解 |

| Classifier 擴充導致 L3→L4 誤升 | 中 | 審計 script + Claude Code judge 逐一確認 |
| 生成的 L4 Q&A 品質不足（泛泛 AI 味） | 高 | 4 維度品質 gate，score <= 2 拒絕 |
| 新 L4 項目 embedding NULL | 低 | keyword search 仍可用；TODO: 後續 rebuild |
| qa_final.json 合併衝突 | 低 | 備份 + atomic write |
| Retrieval 指標退化 | 中 | 每 phase 跑 eval，退化即 revert |
