## Overview

借鑑 karpathy/autoresearch 的 Loop-Forever 模式，讓 agent 在 worktree 中自主循環調整搜尋權重 → 跑 eval → keep/discard。每輪約 40-60 秒，8 小時可跑 400+ 輪。

核心差異：搜尋引擎的 hyperparameter space 約 25 個數值參數（非 neural net），agent 直接閱讀程式碼修改，不需要梯度。

## Architecture Notes (Architect 審查)

### 已採納的設計

1. **eval_local.py 不依賴 Laminar** — 移除 `lmnr.evaluate()` 依賴，純 HTTP + 本地計算
2. **runner.sh 不做 git 操作** — keep/discard 由 agent 執行，保持完全可見性
3. **runner.sh 不管理 server 生命週期** — 需要人工先啟動 `RATE_LIMIT_DEFAULT=9999 pnpm dev`
4. **results.tsv 不追蹤** — `.gitignore` 排除，runtime 產物
5. **Composite score 加權公式固定** — 不可由 agent 修改（等同 prepare.py 不可碰）

### 風險處理

- **tsx watch 重載延遲**：program.md 指示修改後 sleep 5 秒再跑 eval
- **Rate limit**：server 啟動時需 `RATE_LIMIT_DEFAULT=9999`（OPS-001）
- **hit_rate 已 1.0**：composite 加權讓 agent 聚焦 MRR/Precision/NDCG 提升

## Composite Score Formula

```
composite = (
  hit_rate                    × 0.20 +   # 有無相關結果（已 1.0，護欄）
  mrr                         × 0.20 +   # 相關結果排名
  precision_at_k              × 0.15 +   # top-K 精準度
  ndcg_at_k                   × 0.15 +   # 排序品質
  keyword_hit_rate            × 0.15 +   # 關鍵字覆蓋
  multi_label_f1_at_k         × 0.10 +   # 多標籤平衡
  dual_category_recall_at_k   × 0.03 +   # 雙標籤 recall
  boosterless_precision_at_k  × 0.02     # 去 booster 精準度
)
```

Baseline composite ≈ 0.905
