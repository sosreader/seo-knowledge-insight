# Implementation Plan: Health Endpoint Effective LLM

> Generated by /design on 2026-03-20
> Plan file: plans/active/health-effective-llm.md
> Status: PENDING APPROVAL

## Overview

`/health` 的 `capabilities.llm` 從「server 自身能力」改為「本次請求的有效 LLM」。
當 Claude Code 呼叫且 server 無 OpenAI key 時，顯示 `llm: "claude-code"`。
所有 route 內部的 `caps.llm` 決策邏輯不變（仍用 `resolveServerCapabilities()`）。

## Implementation Steps

### Phase 1: TDD 修正（3 步驟）

- [ ] **Step 1**: `api/src/utils/capabilities.ts` — 新增 `resolveHealthCapabilities(userAgent?)`
  - 邏輯：`caller === "claude-code" && llm === "none"` → `llm: "claude-code"`
  - 其餘維度不變，回傳完整 `Capabilities`（含 caller）
  - **不動** `resolveServerCapabilities()` 和 `resolveCapabilities()` — route 決策邏輯零影響

- [ ] **Step 2**: `api/src/routes/health.ts` — 改用 `resolveHealthCapabilities()`
  ```typescript
  // Before
  capabilities: resolveServerCapabilities(),
  // After
  capabilities: resolveHealthCapabilities(c.req.header("user-agent")),
  ```

- [ ] **Step 3**: `api/src/openapi.ts` — health response `llm` enum 加 `"claude-code"`
  - `enum: ["lambda", "local-server"]` for runtime（不變）
  - `enum: ["openai", "claude-code", "none"]` for llm（加一個值）
  - 更新 description 說明語意變更

### Phase 2: 測試（TDD — 先寫）

- [ ] **Step T1**: `api/tests/utils/capabilities.test.ts`
  - Case 1: `resolveHealthCapabilities("claude-code/1.0")` + no OpenAI → `llm: "claude-code"`, `caller: "claude-code"`
  - Case 2: `resolveHealthCapabilities("claude-code/1.0")` + has OpenAI → `llm: "openai"`, `caller: "claude-code"`
  - Case 3: `resolveHealthCapabilities("Mozilla/5.0")` + no OpenAI → `llm: "none"`, `caller: "browser"`
  - Case 4: `resolveHealthCapabilities()` (no UA) → `llm: "none"`, `caller: "unknown"`

- [ ] **Step T2**: `api/tests/routes/health.test.ts`

## Risks & Mitigations

| 風險 | 影響 | 對策 |
|------|------|------|
| Route 邏輯誤用新函式 | 決策分支錯誤 | `resolveServerCapabilities()` 不動，route 不改；新函式命名明確 `Health` |
| OpenAPI enum 不相容 | 前端解析失敗 | `"claude-code"` 是新增值，舊 client 應 fallback |
| Lambda 環境 User-Agent | Lambda 呼叫自身 health 時 UA 不明 | `inferCaller` 已處理 unknown fallback |

## Acceptance Criteria
