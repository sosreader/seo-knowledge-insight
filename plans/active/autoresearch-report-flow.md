## Context

Retrieval 版 autoresearch 跑了 60 輪實驗，但 experiment_log 被刪除、results.tsv 只剩 1 行 baseline，60 次實驗形同沒發生過。Report 版的問題更嚴重：每輪會生成 ~5000 字報告，LLM 有隨機性（相同 prompt 不保證相同輸出），刪除即永久失去。因此需要更嚴格的歸檔機制。

**現有狀態：** `autoresearch/report/` 已建立 v1 版本（program.md + eval_local.py + baseline.json + experiment_log.md + results.tsv + 空 reports/）。本次是重寫 program.md 和補充缺失檔案。

## Implementation Steps

### Phase 1: 檔案修改（5 個檔案，可並行）

- [ ] **Step 1: 重寫 `autoresearch/report/program.md`**
  - 步驟 5: snapshot 輪轉 `round_number % 3`（跳過損壞快照），參照 snapshots.md
  - 步驟 7: 報告寫入 `autoresearch/report/reports/<round>_<snapshot>_pending_<hash>.md`（取代 `/tmp/`），加 `mkdir -p` 確保目錄存在
  - 步驟 11: pending → keep/discard 重命名（**不論結果都保留**）
  - 步驟 12: results.tsv 加 `snapshot` 欄位
  - 步驟 13: experiment_log 條目增加 `Report:` 和 `失敗分類:` 欄位
  - 保留不變：Setup、CAN/CANNOT modify、Diagnostic hints、Exploration strategies、NEVER STOP

- [ ] **Step 2: `autoresearch/report/experiment_log.md` 尾部追加累計分析模板**

- [ ] **Step 3: 新建 `autoresearch/report/snapshots.md`**（4 筆：3 有效 + 1 損壞）

- [ ] **Step 4: `autoresearch/.gitignore` 加入 `report/reports/`**

- [ ] **Step 5: `autoresearch/report/results.tsv` 加 `snapshot` 欄位**（22→23 欄，baseline 行填空）

> 不需要 `.gitkeep` — `reports/` 在 .gitignore 中，由 program.md 的 `mkdir -p` 確保目錄存在。
