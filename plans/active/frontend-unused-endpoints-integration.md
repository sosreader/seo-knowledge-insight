## Phase 4: SSE Streaming Chat（UX 改進・高工作量）

**端點**：`POST /chat/stream`

**修改檔案：**
- `pages/api/seo-insight/[...path].ts` — SSE passthrough（偵測 `text/event-stream` → `ReadableStream` pipe）
- `apis/seoInsight.api.ts` — 新增 `streamChat()` 函式（`fetch` + `getReader()` + SSE 解析）
- `components/admin/seoInsight/StreamingChatMessage.tsx` — 串流訊息泡泡（游標閃爍 + 逐步渲染）
- `pages/admin/seoInsight/chat.tsx` — Streaming toggle + fallback 邏輯

**501 Fallback**：Lambda 回 501 時，顯示一次性通知「Lambda 不支援串流，切換至標準模式」，自動 retry 用 `sendSessionMessage`。

**風險：**
- Next.js Pages Router API Routes 的 SSE streaming 支援有限，需先原型驗證
- SSE 部分 chunk 拆分需維護 buffer，以 `\n\n` 分割完整事件
- streaming → settled 狀態轉換需獨立 `streamingMsg` state，不混入 `messages` 陣列

**驗收標準：**
- [ ] 本地 dev 模式逐 token 渲染
- [ ] Lambda 環境自動 fallback 至 buffered 模式
- [ ] 停止按鈕（AbortController）可中斷串流
- [ ] proxy SSE passthrough 正確轉發 `text/event-stream`
