## Context

現有 Google Search Central Blog fetcher (`scripts/01h_fetch_google_blog.py`) 只抓英文版 (`https://developers.google.com/search/blog`)，已有 255 篇文章。使用者需要加入繁體中文版 (`?hl=zh-tw`) 作為第 10 個外部來源，讓知識庫同時涵蓋 Google 官方 SEO 公告的中英文內容。

## Overview

擴充現有 `01h_fetch_google_blog.py` 加入 `--lang zh-tw` 參數，共享解析邏輯、各自獨立輸出目錄與 source_collection。中文版 slug 加 `zhtw-` 前綴防止衝突。

## 架構決策

**選擇方案 A（擴充 01h），排除 B 和 C：**

| 方案 | 選/排 | 理由 |
|------|-------|------|
| A: 擴充 01h 加 `--lang` | **選擇** | 複用 100% 解析邏輯、單一維護點、`--lang` CLI 分支 |
| B: 獨立新腳本 01k | 排除 | 300+ 行程式碼重複違反 DRY；`importlib.import_module` 增加不必要複雜度 |
| C: 多語言配置檔 | 排除 | 只有 2 語言，YAML/TOML 配置過度設計 |

**可擴展性備註：** 若日後超過 2 語言（如 `ja`、`ko`），改用 `LANG_CONFIG: dict[str, LangConfig]` 映射表替代 `if/else` 分支。目前 2 語言不值得此抽象。

**去重策略：** 中英文視為不同 Q&A（語言不同、繁中語境有額外教學價值）。slug 前綴 `zhtw-` + 獨立目錄確保不衝突。

## Architecture Notes

- **相容性**: 現有 `_discover_blog_posts(since_year=2020)` 呼叫不需改動（`lang` 有預設值 `"en"`）
- **安全性**: SSRF allowlist `developers.google.com` 已涵蓋 zh-tw URL（同域名）；path traversal 防護繼承
- **效能**: zh-tw 版獨立 rate limiting（2s/request），不影響英文版擷取速度
- **API 相容性**: `pipeline.ts` 的 `POST /fetch-articles` 需同步更新（Step 6b），否則 API 端點不會觸發新 fetcher

## Risks & Mitigations

| 風險 | 嚴重度 | 對策 |
|------|--------|------|
| `?hl=zh-tw` 對 scraper 無效 | 中 | Step 8 試跑驗證；若需要加 `Accept-Language: zh-TW` header（RFC 7231 §5.3.5）；01d 已驗證此模式 |
| 僅部分文章有翻譯 | 低 | 預期行為，`_has_sufficient_content()` 過濾空殼頁面 |
| `lang` 參數破壞英文路徑 | 低 | default `"en"` + TDD 回歸測試 |
| slug 衝突 | 低 | `zhtw-` 前綴 + 獨立目錄 + 獨立索引 |
| `api/config.ts` 型別錯誤 | 低 | Step 7 明確更新 + `pnpm test` 型別檢查 |
