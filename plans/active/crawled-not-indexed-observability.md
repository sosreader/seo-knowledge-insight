# Implementation Plan: Crawled-Not-Indexed Observability & Evaluation Integration

> Generated by /design on 2026-03-08
> Status: PENDING APPROVAL

## Overview

將 crawled-not-indexed 功能整合至 Laminar observability 和 eval 體系。目前有完整的 parser/analyzer/evaluator 三層服務 + API endpoint + 報告整合 + 5 筆 golden data，但缺少線上評分（Laminar scoreEvent）、離線 eval executor（Python）、eval group 定義、以及 CI/CD 涵蓋。

## Architectural Decisions

| 決策 | 結論 | 依據 |
|------|------|------|
| 線上評分 sync/async | **Async fire-and-forget**（跟 reports.ts 同 pattern） | 避免 +50-200ms 延遲；Laminar 掛掉不影響 main path |
| Scoring guard | `paths.length > 0` 才評分 | 避免空資料 noise |
| Eval executor 類型 | **API-backed**（POST endpoint） | 與既有 7 個 eval script 一致；測試全 stack |
| API 輸入擴充 | 新增 `raw_tsv` 欄位（與 `source` 擇一） | eval/CI 不需 Google Sheets 依賴 |
| Report evaluator 整合 | **不改 section_coverage 分母**；新增獨立 `has_crawled_not_indexed_section` metric | 避免跨版本 baseline 不可比較 |
| Golden dataset 數量 | 擴充至 **12 筆**（from 5） | 全 deterministic pipeline 不需更多 |
| Python eval for TS feature | **可行** | 既有 6 個 eval 都是 Python 測 TS endpoint |

## Industry & Standards Reference

| 技術決策 | 參照依據 | 類型 | 來源 |
|----------|---------|------|------|
| Fire-and-forget scoring | OpenTelemetry async span export | 業界標準 | OTEL SDK spec |
| API-backed eval executor | Laminar `@evaluate` pattern | 標準化方案 | lmnr-ai/lmnr docs |
| Rule-based evaluator (no LLM) | RAGAS reference-free metrics | 業界最佳實踐 | ragas.io |
| Separate boolean metric vs. changing denominator | Metric orthogonality principle | 業界最佳實踐 | Google SRE Book ch.6 |

## Implementation Steps

### Phase 1: API Schema 擴充（`raw_tsv` 支援）

- [ ] Step 1: 修改 `crawledNotIndexedRequestSchema`，新增 `raw_tsv` 欄位
  - Agent: `tdd-guide`（model: sonnet）
  - File: `api/src/schemas/pipeline.ts`
  - 改動：`source` 改 optional，新增 `raw_tsv: z.string().min(10).max(50000).optional()`，`.refine()` 確保擇一
- [ ] Step 2: 修改 route handler 支援 `raw_tsv` 模式
  - File: `api/src/routes/pipeline.ts`（~475-506 行）
  - 改動：`const data = raw_tsv ? parseCrawledNotIndexedTsv(raw_tsv) : await loadCrawledNotIndexed(source!, tab);`

  - File: `api/tests/routes/pipeline.test.ts`（或新增 test file）
  - 測試：raw_tsv 成功、source+raw_tsv 同時提供回 400、兩者皆空回 400

### Phase 2: Laminar 線上評分

- [ ] Step 4: 在 `POST /crawled-not-indexed` handler 加入 async `observe()` + `scoreEvent()`
  - Agent: `tdd-guide`（model: sonnet）
  - File: `api/src/routes/pipeline.ts`
  - Pattern: 複製 `reports.ts:288-311` 的 fire-and-forget pattern
  - Guard: `if (data.paths.length > 0)` 才評分
  - 6 個 scoreEvents: `cni_path_coverage`, `cni_trend_accuracy`, `cni_has_severity`, `cni_recommendation_specificity`, `cni_data_fidelity`, `cni_overall`
  - 需新增 `import { scoreEvent } from "../utils/laminar-scoring.js";`

### Phase 3: Report Evaluator 整合

- [ ] Step 5: 在 `report-evaluator.ts` 新增 `has_crawled_not_indexed_section` metric

  - 改動：`ReportEvalResult` type 新增 `has_crawled_not_indexed_section: number`，值為 `body.includes("## 檢索未索引分析") ? 1 : 0`
- [ ] Step 6: 在 `reports.ts` 線上評分新增 `scoreEvent("report_has_crawled_not_indexed", ...)`
  - File: `api/src/routes/reports.ts`（~299 行附近）
- [ ] Step 7: 新增 report-evaluator test case（含 `## 檢索未索引分析` 的報告）
  - File: `api/tests/services/report-evaluator.test.ts`
  - 測試：有 section 時 `has_crawled_not_indexed_section === 1`，無時 `=== 0`

### Phase 4: Golden Dataset 擴充 + Eval Group 定義

- [ ] Step 8: 擴充 `eval/golden_crawled_not_indexed.json` 從 5 至 12 筆
  - 新增 7 筆案例：
    1. `crawled_not_indexed_warning_single_path` — 單一路徑 >15% <50%（warning）
    2. `crawled_not_indexed_all_improving_steep` — 所有路徑下降 >20%
    3. `crawled_not_indexed_mixed_critical_improving` — 一個 critical + 多個 improving
    4. `crawled_not_indexed_minimal_paths` — 只有 2 個路徑
    5. `crawled_not_indexed_no_change` — 所有變化率 0%
    6. `crawled_not_indexed_en_hreflang_critical` — /en/ 路徑惡化
    7. `crawled_not_indexed_high_ratio` — 總合/全網域 >50%
- [ ] Step 9: 在 `.claude/evals/pipeline-quality.md` 新增 `crawled_not_indexed_quality` group
  - 5 指標 + 目標值：severity_accuracy >= 0.80, worsening_path_recall >= 0.80, improving_path_recall >= 0.80, path_coverage >= 0.80, overall >= 0.60

## Risks & Mitigations

| Risk | 嚴重度 | Mitigation |
|------|--------|------------|
| `raw_tsv` schema 修改破壞現有呼叫者 | High | `source` 改 optional 但 `.refine()` 確保擇一；既有 URL 模式完全不變 |
| Report evaluator 新 metric 影響 baseline | Low | `has_crawled_not_indexed_section` 是獨立 metric，不改 `section_coverage` |
| CI workflow 需啟動 Node.js server | Medium | 參考既有 eval 的 server start pattern；加 health check timeout |
| Golden TSV 手工格式錯誤 | Medium | 新增前用 `parseCrawledNotIndexedTsv` 驗證每筆可解析 |
| pipeline.ts 超過 800 行 | Low | 目前 ~507 行，加 ~25 行 scoring 仍在限制內 |
