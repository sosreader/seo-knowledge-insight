# Search Console 新推出的 HTTPS 報表透過集合功能整理內容你可以依據偏好儲存及分類內容。
- **發佈日期**: 2022-09-01
- **作者**: Google Search Central
- **來源 URL**: https://developers.google.com/search/blog/2022/09/https-search-console-report?hl=zh-tw
- **來源類型**: article
- **來源集合**: google-search-central-zh

2022 年 9 月 14 日，星期三

我們去年在 Search Console 中推出了[網頁體驗報表](https://support.google.com/webmasters/answer/10218333?hl=zh-tw)，協助網站擁有者瞭解並修正會影響 Google 針對您的網站評估網頁體驗的問題。我們收到的其中一項常見要求，是希望我們針對網站的 HTTPS 狀態提供更多資訊，方便瞭解**哪些**網頁並未透過 HTTPS 提供，以及原因為何。

因應這項要求，未來幾個月我們會在 Search Console 中逐步推出 [HTTPS 報表](https://support.google.com/webmasters/answer/11396518?hl=zh-tw)。您可以透過這份報表深入瞭解，您的 HTTPS 網頁在 Google 搜尋中的呈現情形。此外，這份報表也會列出導致網頁無法以 HTTPS 提供給使用者的問題，並附上網址樣本。

![Search Console 中新推出的 HTTPS 報表](https://developers.google.com/static/search/blog/images/search-console-https-reports.png?hl=zh-tw)

提醒您，「網頁體驗」是一組指標信號，能夠評估使用者對網頁的互動體驗，瞭解純粹網頁資訊價值以外的互動狀態 (行動裝置和電腦裝置都包含在內)。其中包含[網站體驗核心指標](https://web.dev/vitals/?hl=zh-tw#core-web-vitals)，這是一組衡量實際使用者體驗的指標，包括網頁的載入效能、互動性和視覺穩定性，同時也會針對行動裝置相容性、干擾性插頁式廣告和 [HTTPS](https://developers.google.com/search/blog/2016/11/heres-to-more-https-on-web?hl=zh-tw) 提供相關信號。HTTPS 元件可確保使用者的線上安全。

我們希望新增的 HTTPS 報表能幫助您解決導致網頁無法透過 HTTPS 提供的問題，並協助您改善網站訪客的使用體驗。

如果您有任何問題或疑慮，請透過 [Google 搜尋中心社群](https://support.google.com/webmasters/threads?thread_filter=%28category%3Asearch_console%29&hl=zh-tw)或 [Twitter](https://twitter.com/googlesearchc) 與我們聯絡。
