2023 年 4 月 19 日，星期三

這篇網誌文章已發布一段時間。部分資訊可能已經過時，例如圖片可能會有缺漏，連結也可能失效。請參閱有關[網頁體驗](https://developers.google.com/search/docs/appearance/page-experience?hl=zh-tw)的最新說明文件。

實用內容通常可以提供良好的網頁體驗。因此，我們今天在[實用內容製作指南](https://developers.google.com/search/docs/fundamentals/creating-helpful-content?hl=zh-tw)新增了網頁體驗的相關章節，並修訂了[網頁體驗的相關說明頁面](https://developers.google.com/search/docs/appearance/page-experience?hl=zh-tw)。我們相信，這對於網站擁有者在創作內容的過程中，能針對網頁體驗進行更全面的考量。

與先前的指南相比，我們並未新增任何關於網頁體驗的重大內容。如果您有持續關注我們先前討論的內容 (例如 Core Web Vitals)，這些內容將維持不變。

## 簡化網頁體驗指南

多年來，我們的核心排名系統旨在獎勵提供良好網頁體驗的內容，如我們在指南中所述。該指南於 [2011 年](https://developers.google.com/search/blog/2011/05/more-guidance-on-building-high-quality?hl=zh-tw)發布、[2019 年](https://developers.google.com/search/blog/2019/08/core-updates?hl=zh-tw)更新，並在去年加入了「[建立實用、可靠且以使用者為優先的內容](https://developers.google.com/search/docs/fundamentals/creating-helpful-content?hl=zh-tw)」說明頁面。

該說明頁面是[搜尋基礎入門](https://developers.google.com/search/docs/essentials?hl=zh-tw)的重要資源。我們會定期邀請想在 Google 搜尋中取得理想排名的使用者，詳閱自我評估問題和其他相關指南。然而，在該頁面的「內容顯示及製作相關問題」一節中提到了部分網頁體驗相關內容，但有其他部分並未說明。現在，我們改進了這個部分，新增「[提供良好的網頁體驗](https://developers.google.com/search/docs/fundamentals/creating-helpful-content?hl=zh-tw#page-experience)」一節，說明要在 Google 搜尋中取得理想排名應考量的要素。

因此，本節會連結至修訂後的「[瞭解 Google 搜尋結果中的網頁體驗](https://developers.google.com/search/docs/appearance/page-experience?hl=zh-tw)」說明頁面，詳細介紹網頁體驗的作用，以及自我評估問題和資源。該頁面整合了幾個網頁體驗應考量的重要面向，這些內容是我們近幾年一直持續討論的面向，這點未曾改變。

## Search Console 報表

在未來幾個月內，[Search Console](https://search.google.com/search-console/about?hl=zh-tw) 中的網頁體驗報表會轉換為新頁面，該頁面會連結到網頁體驗的一般指南，以及會在 Search Console 中保留的個別 Core Web Vitals 和 HTTPS 報表的資訊主頁檢視畫面。

此外，自 2023 年 12 月 1 日起，我們將淘汰 Search Console 的「行動裝置可用性」報告、[行動裝置相容性測試](https://search.google.com/test/mobile-friendly?hl=zh-tw)工具和 [Mobile-Friendly Test API](https://developers.google.com/webmaster-tools/search-console-api/reference/rest?hl=zh-tw)。這不代表行動裝置可用性對提升 Google 搜尋的排名而言不重要。這對行動裝置使用者來說比以前更為重要，因此，它依然是我們網頁體驗指南中的一部分。不過，自我們[開始推出](https://developers.google.com/search/blog/2014/10/tracking-mobile-usability-in-webmaster?hl=zh-tw)這項報告以來，近十年有許多其他可靠資源可用來評估行動裝置可用性，包括 Chrome 的 [Lighthouse](https://developer.chrome.com/docs/lighthouse/overview?hl=zh-tw)。

整體而言，我們希望藉由這些方式，協助創作者和網站擁有者提供優質的網頁體驗，並在 Google 搜尋中獲得理想排名，進而持續成功吸引訪客。

## 常見問題

### 在沒有網頁體驗報表的情況下，我該如何得知網站是否提供良好的網頁體驗？

網頁體驗報表為一般指南，旨在提供與良好網頁體驗相符的某些指標，而非對所有面向提出全面評估。如要提供良好的網頁體驗，建議採取全面性的做法，包括遵循「[瞭解 Google 搜尋結果中的網頁體驗](https://developers.google.com/search/docs/appearance/page-experience?hl=zh-tw#assess)」頁面中的部分自我評估問題。

### Google 搜尋是否只會使用單項「網頁體驗信號」進行排名？

Google 不會只考量單項信號。我們的核心排名系統會檢視與整體網頁體驗相符的各種信號。

### 網頁體驗信號已列為 Core Web Vitals、行動裝置友善、HTTPS，而且沒有會造成干擾的插頁式廣告。這些信號是否仍會在搜尋排名中使用？

雖然並非所有信號都會直接用於決定排名，但我們發現網頁體驗的所有層面都與搜尋排名密切相關，值得特別留意。

### Core Web Vitals 是否依然重要？

我們強烈建議網站擁有者在 Google 搜尋中獲得良好的 Core Web Vitals，並確保使用者通常皆能享有良好的使用體驗。不過，良好的網頁體驗不只與 Core Web Vitals 相關聯。即便在 Search Console 中的 [Core Web Vitals 報告](https://support.google.com/webmasters/answer/9205520?hl=zh-tw)或第三方網站的 Core Web Vitals 報告中呈現的統計資料良好，也不保證能獲得優秀排名。

### 這對「網頁體驗更新」有何影響？

網頁體驗更新是一項概念，描述網站擁有者應關注的一系列重要網頁體驗面向。具體來說，它[引入了](https://developers.google.com/search/blog/2021/04/more-details-page-experience?hl=zh-tw#details) Core Web Vitals 這項我們核心排名系統納入評估的新信號，以及其他已納入考量的網頁體驗信號，如 HTTPS。這不是獨立的排名系統，也並未將所有信號整合至單一「網頁體驗」信號中。

### 是否應提供良好的網頁體驗，才能顯示在行動裝置的「焦點新聞」輪轉介面中？

網頁體驗不是顯示在「焦點新聞」部分中的資格條件。只要內容符合 [Google 新聞最佳做法](https://support.google.com/news/publisher-center/answer/9607104?hl=zh-tw)和 [Google 新聞政策](https://support.google.com/news/publisher-center/answer/6204050?hl=zh-tw)，我們的自動化系統就會將這類內容納入考量。

### 網頁體驗是根據整個網站還是特定網頁進行評估？

我們的核心排名系統通常是根據特定網頁內容進行評估，包括瞭解與網頁體驗相關的面向。不過，我們也會對部分網站進行全面評估。

### 網頁體驗是否為實用內容系統的考量因素？

實用內容系統主要著重於與內容相關的信號，而非顯示方式和網頁體驗。不過，如同我們的核心排名系統在評估信號時，會納入等同於網頁體驗的考量要素，[實用內容系統](https://developers.google.com/search/updates/helpful-content-update?hl=zh-tw)在某種程度上也是如此。

### 網頁體驗對提升排名有多重要？

Google 搜尋會盡可能顯示最相關的內容，即便網頁體驗差強人意。不過，在許多查詢中，可能有很多網頁都具備實用內容。在這種情況下，如果能提供良好的網頁體驗，就有助於在 Google 搜尋中獲得理想排名。
