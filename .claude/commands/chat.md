# /chat — 互動式 SEO 知識庫問答（不需要 OpenAI）

**你（Claude Code）就是 RAG Chat 引擎**，對每個使用者提問，
先呼叫 qa_tools.py 搜尋知識庫，再結合知識庫內容生成完整回答。

---

## 每輪對話執行步驟

### Step A：搜尋知識庫

```bash
.venv/bin/python scripts/qa_tools.py search --query "<本輪問題>" --top-k 5
```

### Step B：生成回答

依據搜尋結果，按以下結構回答：

```
**回答**：{核心答案，直接切入重點}

**知識庫依據**：
- {Q&A 關鍵句 1}（來源：{source_title}）
- {Q&A 關鍵句 2}（來源：{source_title}）

**行動建議**：
1. {具體可執行的第一步}
2. {具體可執行的第二步}

（若知識庫無直接答案，以「（推論）」標記你的補充）
```

### Step C：維持對話上下文

- 記住本輪對話歷史，後續問題可引用前面的討論
- 若使用者追問，不需要重新搜尋，直接用已有的知識庫結果回答
- 若新問題偏離主題，執行 Step A 重新搜尋

---

## 使用範例

```
/chat
> 我的 Discover 流量最近下降，可能的原因是什麼？
> 你剛才說的 E-E-A-T，如何具體改善？
> 有沒有快速診斷的方法？
```

---

## 注意事項

- 每輪搜尋結果最多使用 5 筆 Q&A，避免 context 過長
- 引用 Q&A 時，務必標示來源（source_title + source_date）
- 若連續 3 輪都搜尋不到相關 Q&A，建議使用者執行 `/extract-qa` 擴充知識庫
